## Consequences

**この決定による影響**

### Positive（利点）

- **パフォーマンス向上**: Redis のインメモリ操作により、セッション取得が高速化
- **セキュリティ強化**: HTTPOnly Cookie により XSS 攻撃からトークンを保護
- **スケーラビリティ**: 複数サーバー間でセッション共有が可能

### Negative（欠点・トレードオフ）

- **インフラコスト**: Redis サーバーの運用コストが追加（月額 $XX）
- **複雑性増加**: セッション管理ロジックがサーバーサイドに移行し、デバッグが複雑化
- **移行コスト**: 既存の localStorage ベースのコードをリファクタリング（見積: XX 時間）

### Risks（リスク）

- **Redis ダウン時の影響**: Redis がダウンすると全セッションが失われる → 対策: Redis Sentinel による高可用性構成
- **セッション肥大化**: セッションデータが大きくなると Redis のメモリを圧迫 → 対策: セッションサイズに上限設定（10KB）
